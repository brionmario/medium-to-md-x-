Integrate the Application

Now that we have a React application, we can integrate the SDK by following the steps mentioned below.

Install the SDK and the dependencies

npm install @asgardeo/auth-react react-router-dom --save
Note: The react-router-dom package is a peer-dependency of the SDK and it is required to be installed for the SDK to work.

Configure the AuthProvider

AuthProvider is used to initialize the SDK and provide the authenticated context to your application.

In the root of the application (index.js), import the provider and wrap the <App> component with it.

The config object in the AuthProvider supports multiple configuration options. Following are the set of minimal configs that are needed for integration.

  • signInRedirectURL: The URL in your application that will receive the authorization code upon user authentication.
  • signOutRedirectURL¬†: The URL in your application where the user is redirected to upon logout.
  • clientID: The OAuth 2.0 Client Identifier valid at the authorization server. (Click here for instructions on obtaining the clientID)
  • serverOrigin: The origin of the Identity Provider. Replace <YOUR_ORGANIZARION> with your actual organization name.
  • scope: Set of scopes that are used to request user attributes such as first_name¬†, last_name etc.

Access the Auth Context

The useAuthContext() hook provided can be used to access the session state. It contains information such as the email address of the authenticated user and the methods that are required for implementing authentication.
